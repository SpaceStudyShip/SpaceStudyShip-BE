📝 현재 문제점
---

- API 설계 스펙 문서가 로컬에만 존재하며 프로젝트 저장소에 포함되어 있지 않음
- 공통 규칙, 인증, Todo, 타이머, 연료, 탐험, 배지, 소셜 등 전 도메인에 걸친 API 스펙이 문서화되어 있으나 팀원이 접근할 수 없는 상태

🛠️ 해결 방안 / 제안 기능
---

- `docs/api-specs/` 경로에 도메인별 API 스펙 마크다운 파일 8개 추가
- 공통 규칙(인증 방식, 응답 형식, 페이지네이션 등) 및 전체 엔드포인트 목록 포함

⚙️ 작업 내용
---

- `docs/api-specs/00_common.md` : 공통 규칙, 전체 32개 엔드포인트 목록
- `docs/api-specs/01_auth.md` : 인증 + 프로필 6개 엔드포인트
- `docs/api-specs/02_todo.md` : Todo + 카테고리 8개 엔드포인트
- `docs/api-specs/03_timer.md` : 타이머 세션 3개 엔드포인트
- `docs/api-specs/04_fuel.md` : 연료 시스템 2개 엔드포인트
- `docs/api-specs/05_exploration.md` : 탐험 4개 엔드포인트
- `docs/api-specs/06_badge.md` : 배지 2개 엔드포인트
- `docs/api-specs/07_social.md` : 친구 + 랭킹 7개 엔드포인트
