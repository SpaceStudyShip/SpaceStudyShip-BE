# ⚙️[기능추가][DevOps] passQL 구조 기반 프로젝트 초기 환경 구축

- **라벨**: 작업전
- **담당자**: Cassiiopeia

---

📝 현재 문제점
---

- `application-dev/prod.yml`이 Git에 추적되어 민감 정보가 노출될 위험 있음
- DB가 MySQL 기반 compose.yaml로 설정되어 있으나 실제 사용 DB는 PostgreSQL
- `application-dev/prod.yml`에 MongoDB 설정이 포함되어 있으나 미사용
- Redis `password` 필드 누락
- CICD 워크플로우의 `PROJECT_NAME`이 다른 프로젝트(`mapsy-back`, `suh-project-utility`)로 고정되어 있음
- `APPLICATION_YML_DIR` 경로가 `MS-Web/...`으로 잘못 설정됨
- Dockerfile이 존재하지 않아 Docker 빌드 불가
- Flyway 미적용으로 DB 스키마 버전 관리 불가

🛠️ 해결 방안 / 제안 기능
---

- `application-dev/prod.yml`을 `.gitignore`에 추가하고 Git 추적 해제
- `SS-Common/build.gradle`에 `flyway-core`, `flyway-database-postgresql` 의존성 추가
- `application-dev/prod.yml`에서 MongoDB 설정 제거, Redis password 추가, Flyway 설정 추가
- DB명 변경: `spacestudyship` → `spacestudyship_app`
- `compose.yaml`을 PostgreSQL 16 + Redis 7 구성으로 교체
- `PROJECT-SPRING-SYNOLOGY-SIMPLE-CICD.yaml` 및 `PR-PREVIEW.yaml`의 프로젝트명·경로·Java 버전 수정
- `Dockerfile` 신규 생성 (`eclipse-temurin:21-jre-alpine` 기반)
- `V0_0_1__init_schema.sql` Flyway 초기 마이그레이션 파일 생성

⚙️ 작업 내용
---

- `SS-Web/src/main/resources/application-dev.yml` 수정 (DB명, MongoDB 제거, Redis password, Flyway)
- `SS-Web/src/main/resources/application-prod.yml` 수정 (MongoDB 제거, Redis password, Flyway)
- `SS-Common/build.gradle` 의존성 추가 (flyway, redis, actuator) 및 MongoDB 제거
- `SS-Web/build.gradle` 중복 의존성 제거
- `SS-Web/src/main/resources/db/migration/V0_0_1__init_schema.sql` 생성
- `compose.yaml` PostgreSQL+Redis로 교체
- `Dockerfile` 생성
- `.github/workflows/PROJECT-SPRING-SYNOLOGY-SIMPLE-CICD.yaml` PROJECT_NAME·경로 수정
- `.github/workflows/PROJECT-SPRING-SYNOLOGY-PR-PREVIEW.yaml` PROJECT_NAME·Java버전·경로 수정
- `SS-Auth/src/main/java/.../auth/service/AuthService.java` Record 접근 방식 컴파일 에러 수정
