📝 현재 문제점
---

- 현재 로그인 요청(`LoginRequest`)에 `fcmToken`, `deviceType`, `deviceId` 필드가 없음
- `user_devices` 테이블과 연동된 Entity/Repository가 존재하지 않음
- 향후 푸시 알림 기능을 위해 FCM 토큰을 디바이스별로 저장해야 함

🛠️ 해결 방안 / 제안 기능
---

- `LoginRequest`에 `fcmToken`, `deviceType`, `deviceId` 필드 추가
- `user_devices` 테이블에 대응하는 `UserDevice` Entity 구현
- 로그인 시 디바이스 정보 upsert (deviceId 기준으로 insert or update)
- 로그아웃 시 해당 디바이스의 FCM 토큰 삭제

⚙️ 작업 내용
---

- `SS-Member/entity/UserDevice` : Entity 추가
- `SS-Member/repository/UserDeviceRepository` : Repository 추가
- `SS-Auth/dto/LoginRequest` : `fcmToken`, `deviceType`, `deviceId` 필드 추가
- `SS-Auth/service/AuthService` : 로그인/로그아웃 시 디바이스 정보 저장/삭제 처리
- Flyway 마이그레이션 : `user_devices` 테이블 생성 SQL 추가
