📝 현재 문제점
---

- 연료 시스템 관련 도메인이 전혀 구현되어 있지 않음
- API 스펙상 연료 잔량 조회, 거래 내역 조회 2개 엔드포인트가 정의되어 있으나 모두 미구현
- 타이머 세션 저장 및 탐험 해금 시 연료를 충전/소비해야 하나 그 기반이 없음

🛠️ 해결 방안 / 제안 기능
---

- `UserFuel` Entity (유저당 1개 잔량 레코드) 및 `FuelTransaction` Entity 구현
- `GET /api/fuel` : 현재 연료 잔량 조회
- `GET /api/fuel/transactions` : 충전/소비 내역 페이지네이션 조회
- 연료 충전/소비는 항상 트랜잭션 내에서 잔량 업데이트 + 거래 내역 insert를 원자적으로 처리
- DB CHECK 제약으로 연료 잔량 음수 방지

⚙️ 작업 내용
---

- `UserFuel` Entity, `FuelTransaction` Entity 구현
- `UserFuelRepository`, `FuelTransactionRepository` 구현
- `FuelService` : `charge()`, `consume()`, `getFuel()`, `getTransactions()` 메서드 구현
- 신규 회원 가입 시 `UserFuel` 초기 레코드 생성 (currentFuel=0)
- `SS-Web/controller/fuel/FuelController` 구현
- Flyway 마이그레이션 : `user_fuel`, `fuel_transactions` 테이블 생성 SQL 추가
